# M1.4 Webhook 加固修复 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 修复 Webhook 接收器在签名兼容性、请求体大小限制、启动配置约束、测试覆盖和配置文档上的问题。

**Architecture:** 保持现有 `receiver + verifier + parser + handler` 结构不变，只做最小闭环修复。验签器兼容 Gitea 原生 hex 与 `sha256=` 前缀格式，接收器对请求体施加固定上限，`serve` 在 secret 未配置时直接失败；同时用 TDD 补齐关键测试并补充 Gitea 配置文档。

**Tech Stack:** Go、Gin、Cobra、标准库 `crypto/hmac` / `crypto/sha256` / `net/http`、Go testing

### Task 1: 修复签名格式兼容性

**Files:**
- Modify: `internal/webhook/verifier.go`
- Test: `internal/webhook/verifier_test.go`

**Step 1: Write the failing test**

在 `internal/webhook/verifier_test.go` 增加测试：

```go
func TestSignatureVerifier_VerifyRawHex(t *testing.T) {
    body := []byte(`{"action":"opened"}`)
    mac := hmac.New(sha256.New, []byte("secret"))
    _, _ = mac.Write(body)
    signature := hex.EncodeToString(mac.Sum(nil))

    verifier := NewSignatureVerifier("secret")
    if err := verifier.Verify(signature, body); err != nil {
        t.Fatalf("Verify() error = %v", err)
    }
}
```

再补：
- `sha256=` 前缀仍可通过
- 非法 hex 失败
- 空 secret 返回 `ErrSecretNotConfigured`
- 空签名返回 `ErrMissingSignature`

**Step 2: Run test to verify it fails**

Run: `go test ./internal/webhook -run 'TestSignatureVerifier'`
Expected: `TestSignatureVerifier_VerifyRawHex` 失败，报签名不匹配。

**Step 3: Write minimal implementation**

在 `internal/webhook/verifier.go` 中实现签名规范化：

```go
func normalizeSignature(signature string) string {
    return strings.TrimPrefix(signature, "sha256=")
}
```

再对规范化后的值做 `hex.DecodeString`，保留原有错误语义与 `hmac.Equal`。

**Step 4: Run test to verify it passes**

Run: `go test ./internal/webhook -run 'TestSignatureVerifier'`
Expected: PASS

**Step 5: Commit**

```bash
git add internal/webhook/verifier.go internal/webhook/verifier_test.go
git commit -m "fix: 兼容 Gitea webhook 签名格式"
```

func TestReceiver_HandlePayloadTooLarge(t *testing.T) {
    gin.SetMode(gin.TestMode)
    router := gin.New()
    RegisterRoutes(router, Config{Secret: "secret", Handler: &stubHandler{}})

    body := strings.Repeat("a", maxPayloadSize+1)
    req := httptest.NewRequest(http.MethodPost, "/webhooks/gitea", strings.NewReader(body))

再补：

Run: `go test ./internal/webhook -run 'TestReceiver'`
Expected: 新增超大 body 用例失败，因为当前未限制大小。

**Step 3: Write minimal implementation**

在 `internal/webhook/receiver.go` 中：

```go

    c.Request.Body = http.MaxBytesReader(c.Writer, c.Request.Body, maxPayloadSize)
    body, err := io.ReadAll(c.Request.Body)
    // 其余逻辑保持不变
}
```

确保读取失败或空 body 都返回 `400`。
