# M3.1 fix.Service 骨架与 Issue 上下文采集 — 设计文档

> 日期：2026-04-02
> 状态：已确认
> 依赖：Phase 1 完成（Webhook / 队列 / Worker / Gitea API / 通知框架）

## 目标

新建 `internal/fix` 包，架构仿照 `internal/review`，实现 Issue 上下文采集，并将 Processor 中的 `fix_issue` 任务从默认 `pool.Run()` 路径改为分发到 `fix.Service`。

**M3.1 不包含**：容器执行（M3.2）、结果解析与回写（M3.3）、fix 专属配置（M3.2）。

## 关键设计决策

| 决策点 | 结论 | 理由 |
|--------|------|------|
| 配置体系 | MVP 最简，直接读全局 `claude.model/effort`，不新增 fix 专属配置段 | 配置需求在 M3.2 prompt 工程阶段才明确 |
| 评论分页 | 单页 50 条，超出 warn | MVP 足够覆盖大部分 bug Issue，后续按需升级 |
| Execute 边界 | 只到上下文采集，不调用容器 | 严格遵循 ROADMAP 里程碑拆分，每个里程碑变更范围小且可独立验证 |
| IssueContext | 纯原始数据承载，不做智能提取 | Claude 更擅长从非结构化文本中提取信息，比写脆弱的正则更可靠 |
| 前瞻性 | 严格最小骨架，不预埋 M3.2/M3.3 占位代码 | YAGNI 原则，Go 加字段是非破坏性变更 |

## 核心类型

### IssueContext（`context.go`）

```go
// IssueContext 通过 Gitea API 采集的 Issue 富上下文（纯原始数据）
type IssueContext struct {
    Issue    *gitea.Issue     // Issue 详情（标题、描述、状态、标签）
    Comments []*gitea.Comment // Issue 评论列表（单页，最多 50 条）
    Labels   []*gitea.Label   // 标签列表（从 Issue.Labels 提取，便于访问）
}
```

### FixResult 与错误类型（`result.go`）

```go
// FixResult 是 Service.Execute 的返回值
type FixResult struct {
    IssueContext *IssueContext // 采集到的 Issue 上下文
    RawOutput    string       // M3.2 补充：Claude CLI 原始 stdout
    CLIMeta      *CLIMeta     // M3.2 补充：CLI 执行元数据（M3.1 始终为 nil）
}

// CLIMeta 执行元数据（独立于 review 包，避免 fix->review 包依赖）
type CLIMeta struct {
    CostUSD    float64
    DurationMs int64
    IsError    bool
    NumTurns   int
    SessionID  string
}

// ErrIssueNotOpen Issue 不处于 open 状态时返回此错误，
// Processor 层据此跳过重试（同 review.ErrPRNotOpen 模式）
var ErrIssueNotOpen = errors.New("issue is not open")
```

CLIMeta 在 fix 包内独立定义（不从 review 包导入），两个包结构相同但独立演化。

## Service 与接口

### 窄接口（`service.go`）

```go
// IssueClient 窄接口，仅暴露 fix 所需的 Gitea API
type IssueClient interface {
    GetIssue(ctx context.Context, owner, repo string, index int64) (*gitea.Issue, *gitea.Response, error)
    ListIssueComments(ctx context.Context, owner, repo string, index int64, opts gitea.ListOptions) ([]*gitea.Comment, *gitea.Response, error)
}

// FixPoolRunner fix 专用的容器执行接口（M3.2 使用，M3.1 声明接口但 Service 构造时可不传入）
type FixPoolRunner interface {
    RunWithCommandAndStdin(ctx context.Context, payload model.TaskPayload, cmd []string, stdinData []byte) (*worker.ExecutionResult, error)
}
```

M3.1 的 IssueClient 不包含 `CreateIssueComment`（M3.3 回写时再扩展）。

### Execute 方法

```go
func (s *Service) Execute(ctx context.Context, payload model.TaskPayload) (*FixResult, error) {
    // 1. 前置校验：issueNum <= 0 返回错误
    // 2. Issue 状态校验：GetIssue，非 open 返回 ErrIssueNotOpen
    // 3. 采集上下文：collectContext（评论单页 50 条，超出 warn）
    // 4. 返回 &FixResult{IssueContext: issueCtx}
    // M3.2 在步骤 3-4 之间插入：构造 prompt + 容器执行
    // M3.3 在步骤 4 之后插入：解析结果 + 回写
}
```

## Processor 集成

### 新增接口与 ProcessorOption（`processor.go`）

```go
// FixExecutor 窄接口，解耦 fix 包
type FixExecutor interface {
    Execute(ctx context.Context, payload model.TaskPayload) (*fix.FixResult, error)
}

// WithFixService 注入 Issue 分析服务
func WithFixService(svc FixExecutor) ProcessorOption {
    return func(p *Processor) { p.fixService = svc }
}
```

Processor 新增 `fixService FixExecutor` 字段。

### 任务路由变更

```go
switch {
case payload.TaskType == model.TaskTypeReviewPR && p.reviewService != nil:
    reviewResult, runErr = p.reviewService.Execute(ctx, payload)
    if reviewResult != nil { result = adaptReviewResult(reviewResult) }
case payload.TaskType == model.TaskTypeFixIssue && p.fixService != nil:
    fixResult, runErr = p.fixService.Execute(ctx, payload)
    if fixResult != nil { result = adaptFixResult(fixResult) }
default:
    result, runErr = p.pool.Run(ctx, payload)
}
```

### ErrIssueNotOpen 处理

在 Processor 错误处理链中新增与 `ErrPRNotOpen` 对称的分支：标记 failed + SkipRetry。

### 通知适配

- `buildStartMessage`：新增 `TaskTypeFixIssue` case，发送 `EventIssueFixStarted`
- `buildNotificationMessage`：现有 `TaskTypeFixIssue` 分支已覆盖，M3.1 无额外元数据

## serve.go 装配层

在现有 reviewOpts 构造块之后：

```go
var fixOpts []queue.ProcessorOption
if deps.GiteaClient != nil {
    fixSvc := fix.NewService(
        deps.GiteaClient,
        fix.WithServiceLogger(slog.Default()),
    )
    fixOpts = append(fixOpts, queue.WithFixService(fixSvc))
}

allOpts := append(reviewOpts, fixOpts...)
processor := queue.NewProcessor(deps.Pool, deps.Store, deps.Notifier, slog.Default(), allOpts...)
```

不修改 `ServiceDeps` 结构体，ProcessorOption 模式天然支持扩展。
