## 1. 目标

从零搭建可编译、可 lint、可跨平台构建的 Go 项目骨架，输出一个带 `version` 子命令的 `dtworkflow` 二进制。

---

## 2. 设计决策记录

| 决策项 | 结论 | 理由 |
|--------|------|------|
| Module path | `otws19.zicp.vip:3000/kelin/dtworkflow` | 与 Gitea 仓库地址一致，自签名证书和端口号不影响 module path 作为标识符使用 |
| Go 版本 | 1.23 | 内置 `log/slog`，`for range` 增强，iterator 支持 |
| 构建工具 | Makefile | Go 社区事实标准，零额外依赖，macOS/Linux 原生支持 |
| golangci-lint | 均衡型（9 个 linter） | 初期不被过多规则拖慢，保证基本正确性和风格一致 |
| Docker 开发环境 | 开发依赖（Redis）+ 生产构建 Dockerfile | Go 开发在宿主机效率最高，Docker 只管外部依赖 |
| 日志库 | 标准库 `log/slog` | 零额外依赖，符合"纯 Go、最少依赖"原则 |
| 配置模板 | 单文件 `configs/dtworkflow.example.yaml` | 初期配置项少，Viper 支持环境变量覆盖 |
| CLI 可运行程度 | 骨架 + version 子命令 | 验证 Cobra + Makefile ldflags 注入的端到端链路 |
| M1.1 依赖 | 仅 Cobra | Viper 留到 M1.8，严格按需引入 |

### 设计要点

- `cmd/dtworkflow/main.go` 只做一件事：调用 `internal/cmd.Execute()`
- Cobra 命令定义放在 `internal/cmd/` 而非 `cmd/` 下，命令注册逻辑是内部实现细节
- `pkg/` 目录 M1.1 暂时为空，用 `.gitkeep` 占位
- `build/` 存放构建相关文件，与 PoC 的 Dockerfile 职责分离

## 6. Makefile 构建系统

### 核心变量

```makefile
APP_NAME    := dtworkflow
MODULE      := otws19.zicp.vip:3000/kelin/dtworkflow
VERSION     := $(shell git describe --tags --always --dirty 2>/dev/null || echo "dev")
GIT_COMMIT  := $(shell git rev-parse --short HEAD 2>/dev/null || echo "unknown")
BUILD_TIME  := $(shell date -u '+%Y-%m-%dT%H:%M:%SZ')
LDFLAGS     := -X '$(MODULE)/internal/cmd.version=$(VERSION)' \
               -X '$(MODULE)/internal/cmd.gitCommit=$(GIT_COMMIT)' \
               -X '$(MODULE)/internal/cmd.buildTime=$(BUILD_TIME)'
```

### Targets

| Target | 说明 |
|--------|------|
| `make build` | 构建本平台二进制到 `bin/` |
| `make build-linux` | 交叉编译 Linux amd64（CGO_ENABLED=0） |
| `make lint` | golangci-lint 静态检查 |
| `make test` | 运行全部测试 |
| `make fmt` | goimports 格式化代码 |
| `make clean` | 清理 `bin/` 目录 |
| `make dev-up` | docker compose 启动开发依赖 |
| `make dev-down` | docker compose 停止开发依赖 |
| `make docker-build` | 构建生产 Docker 镜像 |
| `make help` | 列出所有 target（默认目标） |

### 设计要点

- 构建产物统一输出到 `bin/`（加入 `.gitignore`）
- `make build-linux` 显式 `CGO_ENABLED=0` 确保纯静态编译
- `make help` 作为默认目标
- VERSION 优先从 git tag 获取，无 tag 时回退到 `dev`

## 7. golangci-lint 配置

### 启用的 Linter（9 个）

| 分类 | Linter | 说明 |
|------|--------|------|
| 官方/默认 | `errcheck` | 未处理的 error 返回值 |
| | `govet` | go vet 官方检查 |
| | `staticcheck` | 综合静态分析 |
| | `unused` | 未使用的代码 |
| 正确性 | `gosimple` | 建议更简洁写法 |
| | `ineffassign` | 无效赋值 |
| | `typecheck` | 类型检查 |
| 风格 | `goimports` | import 分组与排序 |
| | `misspell` | 英文拼写错误 |

### 关键配置

- `goimports.local-prefixes`：设为项目 module path，确保内部 import 分组正确
- 测试文件放宽 `errcheck`
- `exclude-use-default: false`：关闭默认排除规则

### 后续可收紧方向（不在 M1.1 范围）

- `errorlint`：error wrapping 规范
- `revive`：灵活代码规范
- `gocritic`：代码审查建议

## 8. Docker / Docker Compose

### 开发环境：`docker-compose.yml`

```yaml
services:
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    command: redis-server --appendonly yes

volumes:
  redis-data:
```

- M1.1 仅 Redis 一个服务，为 M1.5（asynq 任务队列）做准备
- dtworkflow 本身在宿主机 `go run` 开发

### 生产构建：`build/Dockerfile`

```dockerfile
FROM golang:1.23-bookworm AS builder
WORKDIR /src
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
    go build -ldflags "..." -o /out/dtworkflow ./cmd/dtworkflow

FROM debian:bookworm-slim
RUN apt-get update && apt-get install -y --no-install-recommends \
    ca-certificates \
    && rm -rf /var/lib/apt/lists/*
COPY --from=builder /out/dtworkflow /usr/local/bin/
RUN useradd --uid 1001 --create-home dtworkflow
USER dtworkflow
ENTRYPOINT ["dtworkflow"]
```

### 设计要点

- 多阶段构建，最终镜像约 80MB（debian-slim + 二进制 + ca-certificates）
- `CGO_ENABLED=0` 纯静态链接
- 非 root 用户运行（PRD 安全性要求）
- `ca-certificates` 保留：运行时需 HTTPS 访问 Gitea 和 Claude API
- `-ldflags` 由 Makefile 通过 `--build-arg` 传入

### 与 PoC 的关系

- `poc/Dockerfile`：Claude Code Worker 镜像（Node.js + Claude CLI），属于 M1.6
- `build/Dockerfile`：dtworkflow 自身镜像，职责完全不同

## 9. 配置文件模板

`configs/dtworkflow.example.yaml`：

```yaml
# DTWorkflow 配置文件模板
# 复制为 dtworkflow.yaml 后修改

# 服务器配置
server:
  host: "0.0.0.0"
  port: 8080

# Gitea 配置
gitea:
  url: "https://your-gitea-instance.com"
  token: ""
  # 自签名证书时设为 true
  insecure_skip_verify: false

# Redis 配置
redis:
  addr: "localhost:6379"
  password: ""
  db: 0

# 日志配置
log:
  level: "info"    # debug / info / warn / error
  format: "text"   # text / json

# Worker 配置
worker:
  concurrency: 3
  timeout: "30m"
```
