# M4.2 手工集成测试指引

> 日期：2026-04-19
> 作用：M4.2 合入前在测试服务器上执行的端到端验证步骤，覆盖设计文档 §9「E2E 回归」小节。
> 目标仓库：`testorg/javasample`（预设 Maven 项目）。

## 0. 前置

- 测试服务器 SSH 别名：`companytest`（参考 `~/.ssh/config`），部署目录 `/opt/dtworkflow`
- Gitea 测试实例已创建 `testorg/javasample`（含 `backend` 模块与 `pom.xml`）
- 已为 `testorg/javasample` 在 `configs/config.yaml` 下配置：
  ```yaml
  test_gen:
    enabled: true
    module_scope: ["backend"]
    max_retry_rounds: 3
    # review_on_failure: true   # 可选，开启后失败路径也会自动入队 review
  ```
- `worker-full` 镜像已构建并推送，或在服务器本地存在

## 1. Happy Path（PR 创建 + review 触发）

```bash
ssh companytest
cd /opt/dtworkflow
./bin/dtw gen-tests --repo testorg/javasample --module backend --ref main
./bin/dtw wait <task-id>
```

**验证点**：
- [ ] `dtw wait` 最终 status=`succeeded`
- [ ] 容器日志中出现：
  - `gen_tests: fetch + checkout BASE_REF (main)` 前置步骤
  - `gen_tests: checkout -B auto-test/backend origin/main`（首次）或 `origin/auto-test/backend`（复用）
  - Claude 每文件 `commit + push` 记录
- [ ] Gitea 上 `auto-test/backend` 分支存在
- [ ] PR `test: 补充 backend 测试用例` 已自动创建，Base 为 `main`
- [ ] 飞书收到 `EventGenTestsStarted` + `EventGenTestsDone` 两条消息（`failure_category=none`，severity=Info）
- [ ] PR 上出现 DTWorkflow 评论（正文首行含锚点 `<!-- dtworkflow:gen_tests:done -->`）
- [ ] 新 PR 自动触发一次 review 任务（`task_type=review_pr`，`triggered_by=gen_tests:<taskID>`）
- [ ] `test_gen_results` 表新增一行（`task_id` 对应本次任务，`review_enqueued=1`，`output_json` 已截断 ≤ 2 KB）

## 2. 断点续传（容器超时 → asynq retry）

```bash
# 触发一次较大规模的生成
./bin/dtw gen-tests --repo testorg/javasample --module backend --ref main
# 监控容器列表
ssh companytest 'docker ps | grep dtworkflow-worker'
# 在 Claude push 了至少 2~3 个文件后手动 kill 容器
ssh companytest 'docker kill <container-id>'
```

**验证点**：
- [ ] asynq 自动 retry（`./bin/dtw task get <task-id>` 观察 `status=retrying`）
- [ ] 重启容器日志出现 `checkout -B auto-test/backend origin/auto-test/backend`（**分支复用**，不重置）
- [ ] 已 push 的测试文件在重跑后未被删除（git log `origin/auto-test/backend` 连续）
- [ ] 最终 PR 被复用（无新 PR 创建），状态 open

## 3. 污染重置（落后 base / 非 bot author）

```bash
# 手动在 Gitea 上对 auto-test/backend 分支追加一次用户 commit（模拟污染）
# 或让 main 推进 N commits 使 auto-test 落后
./bin/dtw gen-tests --repo testorg/javasample --module backend --ref main
```

**验证点**：
- [ ] 容器日志出现分支重置路径（`checkout -B auto-test/backend <basesha>`）
- [ ] 主动 `git push --force-with-lease=auto-test/backend:<branchsha> origin auto-test/backend`
- [ ] `/tmp/.gen_tests_warnings` 写入 `AUTO_TEST_BRANCH_RESET_PUSHED=1`
- [ ] 飞书 Done 消息包含 Warning 段落（来自 `TestGenOutput.Warnings`）

## 4. Cancel-and-Replace（两次快速触发）

```bash
./bin/dtw gen-tests --repo testorg/javasample --module backend --ref main
# 第一个任务尚未完成时立即发第二个
./bin/dtw gen-tests --repo testorg/javasample --module backend --ref main
```

**验证点**：
- [ ] 第一个任务 `status=cancelled`（由 `TaskCanceller` 取消）
- [ ] 第一个任务对应的容器被 kill
- [ ] 第一个任务的 `auto-test/backend` 远程分支被删除 + PR 被 close（`BranchCleaner` 日志）
- [ ] 第二个任务从头创建新分支 + 新 PR；review 拦截逻辑不误伤

## 5. review 拦截（auto-test/* 在活跃任务期间不入队 review）

在任务 2（断点续传）期间（任务仍 retrying/running）：

```bash
ssh companytest 'curl -s http://<gitea>/api/v1/repos/testorg/javasample/pulls | jq ".[] | {title,state}"'
# 由 entrypoint 在重跑中触发的 push 会产生 PR synchronize webhook
```

**验证点**：
- [ ] `./bin/dtw task list --repo testorg/javasample --type review_pr` 不出现因 synchronize 产生的新 review 任务
- [ ] 服务日志包含 `skip review enqueue for auto-test branch during active gen_tests` 类似信息
- [ ] 任务结束后对 PR 手动 push 会重新正常触发 review

## 6. 回归（不破坏 M4.1）

```bash
./bin/dtw review-pr --repo testorg/javasample --pr <non-auto-test-pr>
./bin/dtw fix-issue --repo testorg/javasample --issue <N>
```

**验证点**：
- [ ] review_pr / fix_issue 路径成功完成
- [ ] 飞书通知 + PR 评论行为与 M4.1 一致（无回归）
