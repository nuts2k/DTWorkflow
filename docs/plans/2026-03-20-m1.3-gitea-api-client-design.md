# M1.3 Gitea API 客户端 - 设计文档

> 日期：2026-03-20
> 状态：待实现

> 依赖：M1.2 CLI 框架（已完成）

---

## 1. 目标

在 `internal/gitea/` 包中实现一个轻量级 Gitea REST API 客户端，覆盖 PR 操作、Issue 操作、仓库操作三类核心 API。客户端基于 `net/http` 直接构建（不依赖第三方 SDK），提供类型安全的方法、结构化错误处理和完整的测试覆盖。交付一个可被后续 Phase 2（PR 评审）和 Phase 3（Issue 修复）直接调用的 Gitea 交互层。

## 2. 设计决策记录

| # | 决策项 | 结论 | 理由 |
|---|--------|------|------|
| 1 | 自建客户端 vs 官方 SDK | **自建轻量客户端**，不依赖 `code.gitea.io/sdk/gitea` | 项目仅需 ~20 个方法（SDK 覆盖 400+）；自建可实现 per-method `context.Context`（SDK 用全局 `SetContext` 是反模式）；直接 JSON 反序列化到自有类型，无映射开销；httptest 测试天然适配；少一个外部依赖 |
| 2 | 服务分层 vs 扁平方法 | **扁平方法**（`client.GetPullRequest()`） | 当前仅 ~20 个方法，分为三组（PR/Issue/Repo）；扁平调用路径更短；如未来方法数显著增长再重构为服务分层（YAGNI） |
| 3 | 客户端构造模式 | **Functional Options**（`NewClient(url, WithToken(...))`) | Go 社区标准模式（Dave Cheney 2014），Gitea SDK、go-gitlab 均采用；扩展性好，向后兼容 |
| 4 | 返回值约定 | **`(data, *Response, error)` 三元组** | 与 go-github 对齐；`*Response` 携带分页元信息（`TotalCount`、`NextPage`）；单资源方法也返回 `*Response` 保持一致性 |
| 5 | 类型定义 | **自有类型**，不复用 SDK 类型 | 只定义项目需要的字段，JSON tag 直接反序列化；消费方（review/fix）不需要 import 第三方包 |
| 6 | context.Context | 每个方法**第一参数**接受 `ctx context.Context` | Go 官方惯例，支持超时取消和链路追踪；避免 SDK 的全局 context 反模式 |
| 7 | 错误处理 | **自定义 `*ErrorResponse`** + 辅助判断函数 | 包含 HTTP 状态码和 API 错误消息；提供 `IsNotFound()`、`IsUnauthorized()` 等便捷函数；支持 `errors.As` 链式检查 |
| 8 | 分页 | **手动分页**，不做自动全量拉取 | 调用方控制分页策略（按需拉取 vs 全量）；提供 `ListOptions` 嵌入 + `*Response` 携带分页信息 |
| 9 | 接口定义 | M1.3 **不定义 interface** | Go 惯例"接口在消费侧定义"；`internal/review/` 等业务包按需声明自己的最小接口；避免预定义过大接口 |
| 10 | 测试策略 | 单元测试用 **httptest**；集成测试用 **build tag** | httptest.NewServer 模拟 Gitea API 响应，零外部依赖；集成测试 `//go:build integration` 隔离，需环境变量指定真实 Gitea |
| 11 | 新增依赖 | **零新增**（仅 Go 标准库） | `net/http` + `encoding/json` + `net/url` 即可；保持依赖最小化 |

## 4. Client 核心设计

### 4.1 Client 结构体

```go
// internal/gitea/client.go

type Client struct {
    baseURL    *url.URL       // Gitea 实例地址，如 https://gitea.example.com
    token      string         // API Token
    httpClient *http.Client   // 可替换的 HTTP 客户端
    userAgent  string         // User-Agent 标识
}
```

### 4.2 Functional Options

```go
type ClientOption func(*Client) error

func NewClient(baseURL string, opts ...ClientOption) (*Client, error) {
    u, err := url.Parse(strings.TrimRight(baseURL, "/"))
    if err != nil {
        return nil, fmt.Errorf("解析 Gitea URL: %w", err)
    }

    c := &Client{
        baseURL:    u,
        httpClient: &http.Client{Timeout: 30 * time.Second},
        userAgent:  "dtworkflow/1.0",
    }

    for _, opt := range opts {
        if err := opt(c); err != nil {
            return nil, fmt.Errorf("应用客户端选项: %w", err)
        }
    }

    if c.token == "" {
        return nil, errors.New("必须提供 API Token：使用 WithToken() 选项")
    }

    return c, nil
}

// WithToken 设置 API Token 认证
func WithToken(token string) ClientOption {
    return func(c *Client) error {
        if token == "" {
            return errors.New("token 不能为空")
        }
        c.token = token
        return nil
    }
}

// WithHTTPClient 替换底层 HTTP 客户端（用于自定义超时、代理、TLS 等）
func WithHTTPClient(hc *http.Client) ClientOption {
    return func(c *Client) error {
        if hc == nil {
            return errors.New("HTTP 客户端不能为 nil")
        }
        c.httpClient = hc
        return nil
    }
}

// WithUserAgent 自定义 User-Agent
func WithUserAgent(ua string) ClientOption {
    return func(c *Client) error {
        c.userAgent = ua
        return nil
    }
}
```

### 4.3 底层请求方法

```go
// newRequest 构造 HTTP 请求，注入认证头和 Content-Type
func (c *Client) newRequest(ctx context.Context, method, path string, body any) (*http.Request, error) {
    u := c.baseURL.JoinPath(path)

    var bodyReader io.Reader
    if body != nil {
        buf, err := json.Marshal(body)
        if err != nil {
            return nil, fmt.Errorf("序列化请求体: %w", err)
        }
        bodyReader = bytes.NewReader(buf)
    }

    req, err := http.NewRequestWithContext(ctx, method, u.String(), bodyReader)
    if err != nil {
        return nil, err
    }

    req.Header.Set("Authorization", "token "+c.token)
    req.Header.Set("Accept", "application/json")
    req.Header.Set("User-Agent", c.userAgent)
    if body != nil {
        req.Header.Set("Content-Type", "application/json")
    }

    return req, nil
}

// doRequest 执行请求，解析响应，返回 *Response
func (c *Client) doRequest(req *http.Request, v any) (*Response, error) {
    httpResp, err := c.httpClient.Do(req)
    if err != nil {
        return nil, fmt.Errorf("HTTP 请求失败: %w", err)
    }
    defer httpResp.Body.Close()

    resp := newResponse(httpResp)

    if err := checkResponse(httpResp); err != nil {
        return resp, err
    }

    if v != nil {
        if err := json.NewDecoder(httpResp.Body).Decode(v); err != nil {
            return resp, fmt.Errorf("解析响应: %w", err)
        }
    }

    return resp, nil
}

// doRequestRaw 执行请求，返回原始字节（用于 diff/patch 等非 JSON 响应）
func (c *Client) doRequestRaw(req *http.Request) ([]byte, *Response, error) {
    httpResp, err := c.httpClient.Do(req)
    if err != nil {
        return nil, nil, fmt.Errorf("HTTP 请求失败: %w", err)
    }
    defer httpResp.Body.Close()

    resp := newResponse(httpResp)

    if err := checkResponse(httpResp); err != nil {
        return nil, resp, err
    }

    data, err := io.ReadAll(httpResp.Body)
    if err != nil {
        return nil, resp, fmt.Errorf("读取响应体: %w", err)
    }

    return data, resp, nil
}
```

### 4.4 查询参数编码

```go
// addListOptions 将 ListOptions 编码为 URL 查询参数
func addListOptions(path string, opts ListOptions) string {
    params := url.Values{}
    if opts.Page > 0 {
        params.Set("page", strconv.Itoa(opts.Page))
    }
    if opts.PageSize > 0 {
        params.Set("limit", strconv.Itoa(opts.PageSize))
    }
    if len(params) == 0 {
        return path
    }
    return path + "?" + params.Encode()
}
```

## 5. 类型定义

### 5.1 设计原则

- 只定义项目实际需要的字段，不追求覆盖 API 全部字段
- 使用 `json` tag 直接从 Gitea API JSON 反序列化
- 后续 Phase 需要新字段时直接添加（struct 加字段是向后兼容的）
- 指针字段仅用于可选/可空值（`*User`），基础类型用零值

```go
func (c *Client) GetPullRequest(ctx context.Context, owner, repo string, index int64) (*PullRequest, *Response, error) {
    path := fmt.Sprintf("/api/v1/repos/%s/%s/pulls/%d", owner, repo, index)
    req, err := c.newRequest(ctx, http.MethodGet, path, nil)
    if err != nil {
        return nil, nil, err
    }
    var pr PullRequest
    resp, err := c.doRequest(req, &pr)
    if err != nil {
        return nil, resp, err
    }
    return &pr, resp, nil
}

func (c *Client) GetPullRequestDiff(ctx context.Context, owner, repo string, index int64) (string, *Response, error) {
    path := fmt.Sprintf("/api/v1/repos/%s/%s/pulls/%d.diff", owner, repo, index)
    req, err := c.newRequest(ctx, http.MethodGet, path, nil)
    if err != nil {
        return "", nil, err
    }
    // diff 端点返回纯文本，不是 JSON
    req.Header.Set("Accept", "text/plain")
    data, resp, err := c.doRequestRaw(req)
    if err != nil {
        return "", resp, err
    }
    return string(data), resp, nil
}
```

## 7. 错误处理

### 7.1 ErrorResponse

```go
// internal/gitea/errors.go

// ErrorResponse Gitea API 返回的错误
type ErrorResponse struct {
    Response *http.Response `json:"-"`
    Message  string         `json:"message"`
    URL      string         `json:"url,omitempty"`
}

func (e *ErrorResponse) Error() string {
    return fmt.Sprintf("%s %s: %d %s",
        e.Response.Request.Method,
        e.Response.Request.URL.Path,
        e.Response.StatusCode,
        e.Message,
    )
}

// checkResponse 检查 HTTP 状态码，非 2xx 时解析错误体
func checkResponse(r *http.Response) error {
    if r.StatusCode >= 200 && r.StatusCode < 300 {
        return nil
    }
    errResp := &ErrorResponse{Response: r}
    data, err := io.ReadAll(r.Body)
    if err == nil && len(data) > 0 {
        _ = json.Unmarshal(data, errResp)
    }
    if errResp.Message == "" {
        errResp.Message = http.StatusText(r.StatusCode)
    }
    return errResp
}
```

### 7.2 辅助判断函数

```go
// IsNotFound 判断是否为 404 错误
func IsNotFound(err error) bool {
    return hasStatusCode(err, http.StatusNotFound)
}

// IsUnauthorized 判断是否为 401 错误
func IsUnauthorized(err error) bool {
    return hasStatusCode(err, http.StatusUnauthorized)
}

// IsForbidden 判断是否为 403 错误
func IsForbidden(err error) bool {
    return hasStatusCode(err, http.StatusForbidden)
}

// IsConflict 判断是否为 409 错误（如分支已存在）
func IsConflict(err error) bool {
    return hasStatusCode(err, http.StatusConflict)
}

func hasStatusCode(err error, code int) bool {
    var errResp *ErrorResponse
    if errors.As(err, &errResp) {
        return errResp.Response.StatusCode == code
    }
    return false
}
```

## 8. Gitea API 已知限制与应对

调研中发现的 Gitea API 限制，记录在此供后续 Phase 开发参考：

| 限制 | 影响 | 应对策略 |
|------|------|----------|
| 创建 Review 时必须一次性提交所有 inline comments，无法对已提交 Review 追加评论（Issue #36300） | Phase 2 评审时需收集所有问题后一次性提交 | `CreatePullReview` 在 `Comments` 字段批量传入所有行级评论 |
| `pull_request_review_comment` webhook 事件不成熟（Issue #26023） | M1.4 Webhook 设计时需注意 | 使用 `pull_request_review` 事件替代 |
| diff 端点返回纯文本（非 JSON） | 需要单独的 raw 响应处理 | 已通过 `doRequestRaw` 方法支持 |
| 文件内容 API 返回 base64 编码 | 调用方需解码 | 在 `GetFileContent`（raw 端点）中直接返回解码后的字节 |

## 9. 文件组织

```
internal/gitea/
├── client.go              # Client struct、NewClient、functional options、newRequest/doRequest
├── errors.go              # ErrorResponse、checkResponse、IsNotFound 等辅助函数
├── types.go               # 所有实体类型和请求选项类型
├── response.go            # Response struct、分页解析
├── pulls.go               # PR 操作方法（8 个）
├── issues.go              # Issue 操作方法（7 个）
├── repos.go               # 仓库操作方法（5 个）
├── client_test.go         # Client 构造、newRequest、checkResponse 测试
├── pulls_test.go          # PR 方法单元测试
├── issues_test.go         # Issue 方法单元测试
├── repos_test.go          # 仓库方法单元测试
├── testutil_test.go       # 测试辅助函数（setup、testMethod、loadFixture 等）
├── integration_test.go    # 集成测试（//go:build integration）
└── testdata/              # JSON fixture 文件
    ├── pull_request.json
    ├── pull_requests.json
    ├── pull_review.json
    ├── changed_files.json
    ├── issue.json
    ├── issues.json
    ├── comment.json
    ├── comments.json
    ├── labels.json
    ├── repo.json
    ├── branch.json
    └── contents.json
```

### 设计要点

- **一个文件一个资源域**：`pulls.go`、`issues.go`、`repos.go` 分别包含对应的全部方法
- **类型集中定义**：`types.go` 包含所有实体和选项类型，避免跨文件查找
- **错误独立文件**：`errors.go` 专注错误处理，与业务方法分离
- **测试文件与源文件对应**：`pulls.go` → `pulls_test.go`
- **fixture 文件**：`testdata/` 目录存放 JSON 响应模板，与测试代码分离

## 10. 测试策略

### 10.1 单元测试（httptest）

每个 API 方法至少一个正常路径测试和一个错误路径测试。使用 `httptest.NewServer` 模拟 Gitea API。

#### 测试辅助函数

```go
// internal/gitea/testutil_test.go

// setup 创建 mock HTTP 服务器和指向它的 Client
func setup(t *testing.T) (*http.ServeMux, *Client) {
    t.Helper()
    mux := http.NewServeMux()
    server := httptest.NewServer(mux)
    t.Cleanup(server.Close)

    client, err := NewClient(server.URL, WithToken("test-token"))
    if err != nil {
        t.Fatalf("创建测试客户端: %v", err)
    }
    return mux, client
}

// testMethod 验证 HTTP 方法
func testMethod(t *testing.T, r *http.Request, want string) {
    t.Helper()
    if r.Method != want {
        t.Errorf("HTTP 方法 = %s, 期望 %s", r.Method, want)
    }
}

// testHeader 验证请求头
func testHeader(t *testing.T, r *http.Request, name, want string) {
    t.Helper()
    if got := r.Header.Get(name); got != want {
        t.Errorf("Header %s = %q, 期望 %q", name, got, want)
    }
}

// loadFixture 读取 testdata 目录下的 JSON fixture
func loadFixture(t *testing.T, name string) []byte {
    t.Helper()
    data, err := os.ReadFile(filepath.Join("testdata", name))
    if err != nil {
        t.Fatalf("加载 fixture %s: %v", name, err)
    }
    return data
}

// writeJSON 向 ResponseWriter 写入 JSON
func writeJSON(w http.ResponseWriter, data []byte) {
    w.Header().Set("Content-Type", "application/json")
    w.Write(data)
}
```

#### 测试示例

```go
// internal/gitea/pulls_test.go

func TestGetPullRequest(t *testing.T) {
    mux, client := setup(t)

    mux.HandleFunc("/api/v1/repos/owner/repo/pulls/42", func(w http.ResponseWriter, r *http.Request) {
        testMethod(t, r, "GET")
        testHeader(t, r, "Authorization", "token test-token")
        writeJSON(w, loadFixture(t, "pull_request.json"))
    })

    pr, resp, err := client.GetPullRequest(context.Background(), "owner", "repo", 42)
    if err != nil {
        t.Fatalf("GetPullRequest 失败: %v", err)
    }
    if pr.Number != 42 {
        t.Errorf("PR Number = %d, 期望 42", pr.Number)
    }
    if resp.Response.StatusCode != http.StatusOK {
        t.Errorf("StatusCode = %d, 期望 200", resp.Response.StatusCode)
    }
}

func TestGetPullRequest_NotFound(t *testing.T) {
    mux, client := setup(t)

    mux.HandleFunc("/api/v1/repos/owner/repo/pulls/999", func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusNotFound)
        writeJSON(w, []byte(`{"message":"pull request not found"}`))
    })

    _, _, err := client.GetPullRequest(context.Background(), "owner", "repo", 999)
    if !IsNotFound(err) {
        t.Errorf("期望 NotFound 错误, 得到: %v", err)
    }
}

func TestCreatePullReview_WithInlineComments(t *testing.T) {
    mux, client := setup(t)

    mux.HandleFunc("/api/v1/repos/owner/repo/pulls/42/reviews", func(w http.ResponseWriter, r *http.Request) {
        testMethod(t, r, "POST")
        // 验证请求体包含 inline comments
        var opts CreatePullReviewOptions
        json.NewDecoder(r.Body).Decode(&opts)
        if len(opts.Comments) != 2 {
            t.Errorf("期望 2 条 inline comment, 得到 %d", len(opts.Comments))
        }
        writeJSON(w, loadFixture(t, "pull_review.json"))
    })

    review, _, err := client.CreatePullReview(context.Background(), "owner", "repo", 42, CreatePullReviewOptions{
        State: ReviewStateRequestChanges,
        Body:  "发现 2 个问题需要修复",
        Comments: []ReviewComment{
            {Path: "main.go", Body: "空指针风险", NewLineNum: 15},
            {Path: "util.go", Body: "SQL 注入", NewLineNum: 32},
        },
    })
    if err != nil {
        t.Fatalf("CreatePullReview 失败: %v", err)
    }
    if review.ID == 0 {
        t.Error("期望 Review ID 非零")
    }
}
```

### 10.3 集成测试

```go
// internal/gitea/integration_test.go
//go:build integration

package gitea

// 需要环境变量：
//   GITEA_URL   - Gitea 实例地址（如 https://gitea.example.com）
//   GITEA_TOKEN - API Token
//   GITEA_REPO  - 测试仓库（如 testowner/testrepo）

func TestIntegration_GetRepo(t *testing.T) {
    client := newIntegrationClient(t)
    owner, repo := splitRepo(t, os.Getenv("GITEA_REPO"))

    repository, _, err := client.GetRepo(context.Background(), owner, repo)
    if err != nil {
        t.Fatalf("GetRepo: %v", err)
    }
    if repository.FullName == "" {
        t.Error("期望 FullName 非空")
    }
    t.Logf("仓库: %s (默认分支: %s)", repository.FullName, repository.DefaultBranch)
}

func newIntegrationClient(t *testing.T) *Client {
    t.Helper()
    giteaURL := os.Getenv("GITEA_URL")
    giteaToken := os.Getenv("GITEA_TOKEN")
    if giteaURL == "" || giteaToken == "" {
        t.Skip("跳过集成测试：需设置 GITEA_URL 和 GITEA_TOKEN")
    }
    client, err := NewClient(giteaURL, WithToken(giteaToken))
    if err != nil {
        t.Fatalf("创建客户端: %v", err)
    }
    return client
}
```


| 依赖 | 说明 |

### M1.3 不做的事

- 不实现 Webhook 接收和事件解析（M1.4）
- 不实现自动重试/速率限制/断路器（内网场景暂不需要）
- 不实现 Gitea 管理 API（用户管理、组织管理等）
- 不定义消费侧 interface（由消费方在 Phase 2/3 按需定义）
- 不做 OAuth2 认证（仅 Token 认证，符合当前需求）

## 13. 实现顺序建议

```
Step 1: 骨架
  client.go     → Client struct + NewClient + Options + newRequest/doRequest/doRequestRaw
  errors.go     → ErrorResponse + checkResponse + IsNotFound 等
  types.go      → 所有类型定义
  response.go   → Response struct + 分页解析

Step 2: 仓库操作（最基础）
  repos.go      → GetRepo, GetBranch, CreateBranch, GetContents, GetFileContent
  repos_test.go → 对应测试

Step 3: PR 操作（Phase 2 核心）
  pulls.go      → 8 个方法
  pulls_test.go → 对应测试

Step 4: Issue 操作（Phase 3 核心）
  issues.go     → 7 个方法
  issues_test.go → 对应测试

Step 5: 集成测试 + 清理
  integration_test.go → 连通性验证
  testutil_test.go    → 公共测试辅助
```
